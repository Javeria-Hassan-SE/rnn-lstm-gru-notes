Why Do We Need Sequence Memory?

🧠 The Problem with Normal Neural Networks

A standard neural network — also called a Feedforward Neural Network — processes each input independently. It has no concept of "what came before." Imagine you are reading the sentence:

"The cat sat on the mat because it was tired."

To understand what "it" refers to, you need to remember the word "cat" from earlier in the sentence. A standard neural network cannot do this — it processes "it" without any memory of the previous words.

This is the core challenge of sequential data: data where order and context matter. Examples include:

  • Language and text (the meaning of a word depends on what came before)
  • Music (each note makes sense relative to the melody so far)
  • Stock prices (today's price depends on recent trends)
  • Weather (tomorrow's forecast depends on the past week)
  • Video frames (each frame connects to previous frames)

Recurrent Neural Networks

📐 What is a Recurrent Neural Network?

A Recurrent Neural Network (RNN) is a type of neural network designed specifically for sequential data. The word "recurrent" means "occurring repeatedly" — and that is exactly what happens: the network loops over each element in the sequence, maintaining a hidden state that acts like a memory.

Think of it like reading a book. When you read word by word, you carry the context of everything you have read so far in your mind. As you read each new word, you update your understanding. An RNN does the same: at each time step, it takes the current input AND its previous memory, and produces a new memory state.

RNN Unrolled Through Time
↑ Each cell shares the same weights W — the network is "unrolled" for visualization. In reality, it's one cell processing elements one by one.
🔢 The Core RNN Equations (Explained Step by Step)

At every time step t, the RNN performs exactly two calculations:

Equation 1 — Hidden State Update hₜ = tanh( Wₕ·hₜ₋₁ + Wₓ·xₜ + b )
Equation 2 — Output yₜ = Wᵧ·hₜ + bᵧ
Symbol Full Name What It Means
hₜHidden State at time tThe "memory" of the network at the current step
hₜ₋₁Previous Hidden StateThe memory from the last step — carries past information
xₜInput at time tThe current input (e.g., the current word or stock price)
WₕRecurrent Weight MatrixLearned weights for combining past memory
WₓInput Weight MatrixLearned weights for processing new input
bBiasAdjustment term, helps the network learn offsets
tanhHyperbolic TangentActivation function — squishes values to range [-1, 1]
yₜOutput at time tThe prediction or output at this step
🔢 Numerical Walkthrough: RNN Step by Step

Let's trace through an RNN with tiny numbers to see exactly what happens. We'll predict the next value in a simple sequence: [1, 2, 3, ?]

Setup: h₀ = 0 (no initial memory)  |  Wₓ = 0.5 (input weight)  |  Wₕ = 0.8 (recurrent weight)  |  b = 0.1

STEP t=1
Input: x₁ = 1  |  Previous hidden: h₀ = 0
h₁ = tanh( 0.5×1 + 0.8×0 + 0.1 )
h₁ = tanh( 0.5 + 0 + 0.1 ) = tanh(0.6)
h₁ = 0.537 ← Network now remembers seeing "1"
STEP t=2
Input: x₂ = 2  |  Previous hidden: h₁ = 0.537
h₂ = tanh( 0.5×2 + 0.8×0.537 + 0.1 )
h₂ = tanh( 1.0 + 0.43 + 0.1 ) = tanh(1.53)
h₂ = 0.909 ← Now carries memory of both 1 and 2
STEP t=3
Input: x₃ = 3  |  Previous hidden: h₂ = 0.909
h₃ = tanh( 0.5×3 + 0.8×0.909 + 0.1 )
h₃ = tanh( 1.5 + 0.727 + 0.1 ) = tanh(2.327)
h₃ = 0.981 ← Compressed memory of the full sequence [1,2,3]
OUTPUT
y = Wᵧ × h₃ = 2.0 × 0.981
ŷ = 1.962 ≈ 2  (After training with more data, weights adjust to predict ~4)
Note: Real prediction accuracy comes after backpropagation training, not from these initial weights

⚠️ The Vanishing Gradient Problem

The biggest weakness of vanilla RNNs is called the Vanishing Gradient Problem. Here is what happens during training:

When we train a neural network, we use a process called Backpropagation Through Time (BPTT). This means we calculate how wrong the network was, and then send that error signal backwards through all the time steps to adjust the weights.

The Problem: As the error signal travels backwards through many time steps, it gets multiplied by the weight matrix at each step. If those weights are small (less than 1), the gradient gets exponentially smaller — it "vanishes" before reaching the early time steps. The network cannot learn long-range dependencies.

This is why Long Short-Term Memory (LSTM) and Gated Recurrent Unit (GRU) networks were invented — they solve this problem with special "gate" mechanisms.

↩️ Backpropagation Through Time (BPTT) — Explained

Backpropagation Through Time is how RNNs learn. In a regular neural network, we send the error backwards layer by layer. In an RNN, we "unroll" the network through time and send the error backwards through each time step.

  1. Run the RNN forward through the full sequence (forward pass)
  2. Calculate the loss (how wrong was the final prediction?)
  3. Send the error backwards through each time step (backward pass)
  4. At each step, compute how much each weight contributed to the error
  5. Update weights to reduce the error

Long Short-Term Memory Networks

🏛️ What is LSTM?

Long Short-Term Memory (LSTM) was invented by Sepp Hochreiter and Jürgen Schmidhuber in 1997. It was a breakthrough solution to the vanishing gradient problem.

The key innovation: instead of a single hidden state, LSTM has two separate tracks of information:

CELL STATE (Cₜ)
The "long-term memory". Runs straight through with only minor linear interactions — gradients flow easily. Think of it as a conveyor belt.
HIDDEN STATE (hₜ)
The "short-term memory" or working memory. Passed between time steps and used for output predictions.
🚪 The Four Gates of LSTM — The Heart of the Architecture

LSTM uses gates to control what information flows through. A gate is simply a sigmoid activation (outputs 0 to 1) multiplied by the information — 0 means "block completely", 1 means "let everything through".

🗑️ Forget Gate ✍️ Input Gate 🧮 Candidate Gate 📤 Output Gate
① FORGET GATE — fₜ
fₜ = σ( Wf · [hₜ₋₁, xₜ] + bf )

Question it asks: "What old information should I throw away?"

Uses a sigmoid function (outputs 0 to 1). Multiplied element-wise with the previous cell state. 0 = completely forget, 1 = completely keep.

② INPUT GATE — iₜ
iₜ = σ( Wi · [hₜ₋₁, xₜ] + bi )

Question it asks: "What new information should I store?"

Decides which new values to update. Works together with the Candidate Gate to decide what gets written into the cell state.

③ CANDIDATE GATE — C̃ₜ
C̃ₜ = tanh( Wc · [hₜ₋₁, xₜ] + bc )

Creates a vector of candidate values that could be added to the cell state. Uses tanh to keep values between [-1, 1]. The input gate then filters which of these candidates are actually added.

④ OUTPUT GATE — oₜ
oₜ = σ( Wo · [hₜ₋₁, xₜ] + bo )

Question it asks: "What should I output?"

Determines what part of the cell state becomes the new hidden state (which is also the output).

The Complete Cell State Update (putting it all together):

Cell State Update Cₜ = fₜ ⊙ Cₜ₋₁ + iₜ ⊙ C̃ₜ
New Hidden State hₜ = oₜ ⊙ tanh(Cₜ)

⊙ means element-wise multiplication (Hadamard product). The forget gate "erases" parts of the old cell state, and the input gate "writes" new information in.

🔢 Numerical LSTM Example: Remembering a Person's Name

Scenario: The network reads "Alice is a doctor. She later became..." — at "She", the network must remember "Alice" (a female). Let's trace the LSTM gates.

INPUT
xₜ = "She"  |  hₜ₋₁ = [0.8, 0.2, 0.9] (encodes: female name, past tense, subject)
Cₜ₋₁ = [0.7, 0.1, 0.8] (long-term: Alice remembered in cell state)
FORGET
fₜ = σ(Wf · [hₜ₋₁, xₜ]) ≈ [0.9, 0.1, 0.95]
Interpretation: Keep 90% of "Alice is female" (index 0), forget 90% of "past tense" (index 1), keep 95% of "is subject" (index 2)
After forget: Cₜ₋₁ × fₜ = [0.63, 0.01, 0.76]
INPUT
iₜ = σ(...) ≈ [0.05, 0.9, 0.1]   C̃ₜ = tanh(...) ≈ [0.1, 0.8, 0.5]
New info to add: iₜ × C̃ₜ = [0.005, 0.72, 0.05]
Interpretation: Write in new info: "She is a pronoun referring to a person" (index 1 activated strongly)
CELL
Cₜ = [0.63, 0.01, 0.76] + [0.005, 0.72, 0.05]
Cₜ = [0.635, 0.73, 0.81] ← Cell state now holds: Alice's gender, pronoun reference, subject role
OUTPUT
oₜ = σ(...) ≈ [0.8, 0.6, 0.9]
hₜ = oₜ × tanh(Cₜ) = [0.8, 0.6, 0.9] × [0.56, 0.62, 0.67]
hₜ = [0.448, 0.372, 0.603] ← Hidden state ready to help predict the next word
LSTM Cell — Internal Architecture
✅ Why LSTM Solves the Vanishing Gradient Problem

The key insight is the cell state pathway. Because the cell state only undergoes element-wise multiplication and addition (not matrix multiplications through many layers), gradients can flow back through hundreds of time steps without shrinking to near-zero.

The Cell State is like a highway for gradients. During backpropagation, gradients flow through this highway with little obstruction. The gates allow the network to learn to "protect" important gradients from vanishing.

The forget gate is especially important here — when the forget gate is set to 1 for a particular dimension, that dimension's gradient flows back perfectly unchanged. This enables LSTMs to learn dependencies across hundreds or even thousands of time steps.

Gated Recurrent Units

⚡ What is a GRU?

Gated Recurrent Unit (GRU) was introduced by Kyunghyun Cho et al. in 2014. It is a simplified version of LSTM that achieves similar performance but with fewer parameters and faster training.

GRU makes two key simplifications compared to LSTM:

  • Combines the forget gate and input gate into a single Update Gate
  • Merges the cell state and hidden state into a single hidden state

The result: GRU uses 2 gates instead of LSTM's 4, fewer parameters, trains faster, and works just as well on many tasks.

📐 GRU Architecture and Equations
① RESET GATE — rₜ
rₜ = σ( Wr · [hₜ₋₁, xₜ] )

Controls how much of the past to forget when creating the candidate hidden state. When rₜ ≈ 0, it ignores the previous state entirely (like starting fresh).

② UPDATE GATE — zₜ
zₜ = σ( Wz · [hₜ₋₁, xₜ] )

Controls how much to update the hidden state. When zₜ ≈ 1, keep old state (like LSTM's forget gate keeping everything). When zₜ ≈ 0, replace with new information.

Candidate Hidden State h̃ₜ = tanh( W · [rₜ ⊙ hₜ₋₁, xₜ] )
Final Hidden State hₜ = (1 - zₜ) ⊙ hₜ₋₁ + zₜ ⊙ h̃ₜ
Elegant design: The final state update is a linear interpolation between the old state and the candidate. When zₜ = 0, it keeps the old state. When zₜ = 1, it fully adopts the new candidate. This single gate replaces both forget and input gates of LSTM!
⚖️ RNN vs LSTM vs GRU — Head-to-Head Comparison
Feature Vanilla RNN LSTM GRU
Number of Gates 0 4 gates 2 gates
Memory States 1 (hidden state h) 2 (cell state C + hidden h) 1 (hidden state h)
Long-term Dependencies Poor Excellent Very Good
Training Speed Fastest Slowest Fast
Number of Parameters Fewest Most (4× RNN) Moderate (3× RNN)
Vanishing Gradient Severe problem Solved Largely solved
Best For Simple short sequences, learning Long sequences, complex tasks, NLP Medium sequences, when training speed matters
Real Use Cases Toy examples, baseline Machine translation, speech Time series, medium-length sequences
💡
Rule of Thumb: In practice, try GRU first — it trains faster and often performs similarly to LSTM. If performance is not good enough on very long sequences (thousands of steps), switch to LSTM. Vanilla RNN is mainly used for educational purposes today.
🧠 Quick Check: Which gate in an LSTM is responsible for deciding what to "forget" from the previous cell state?
✓ Correct! The Forget Gate (fₜ) uses a sigmoid activation to output values between 0 and 1. These are multiplied with the previous cell state: 0 = completely forget, 1 = keep entirely. This is how LSTM decides what long-term memories are no longer relevant.

Time Series Forecasting with RNNs

📈 What is Time Series Data?

A time series is a sequence of data points collected or indexed in chronological (time) order. Each data point is associated with a timestamp. The key characteristic: the order matters, and values at one time are often influenced by values at previous times.

FINANCIAL
Stock prices, exchange rates, cryptocurrency values, trading volumes
NATURAL
Temperature, rainfall, seismic activity, solar energy output
INDUSTRIAL
Sensor readings, power consumption, machine vibrations
🔧 Key Concepts in Time Series Forecasting
Window Size / Look-back Period
How many past time steps the model uses to make its prediction. Example: Use the last 60 days of stock prices to predict the next day.
Stationarity
A stationary series has constant mean and variance over time. Many time series methods require stationarity. Differencing (subtracting previous value) can make a series stationary.
Train/Validation/Test Split
For time series, you MUST split chronologically — not randomly! Future data cannot influence past predictions. Typically: 70% train, 10% validation, 20% test.
Normalization
Scale data to [0,1] or [-1,1] using Min-Max scaling or Standard scaling. Essential for neural networks to train effectively on financial data.
Min-Max Normalization Formula x_normalized = (x - x_min) / (x_max - x_min) → Result always in [0, 1]
🏗️ Data Preparation Pipeline for RNN

This is the most important (and most overlooked) part of any time series project. Here is the full pipeline:

STEP 1
Collect Raw Data — Download historical price data (open, high, low, close, volume)
STEP 2
Handle Missing Values — Fill gaps caused by holidays, trading halts, etc. Forward-fill is common for financial data
STEP 3
Feature Engineering — Add technical indicators (moving averages, RSI, MACD), candlestick pattern flags
STEP 4
Normalize/Scale — Apply Min-Max or Standard Scaler to bring all values to similar ranges
STEP 5
Create Sliding Windows — Convert the series into (input_sequence, target) pairs using a sliding window of fixed size
STEP 6
Chronological Split — Split into train, validation, and test sets in time order
STEP 7
Reshape for RNN — Input shape must be (samples, timesteps, features)

Code Lab: From Scratch to Stock Predictor

🔵 Building a Vanilla RNN from Scratch (NumPy)

Before using frameworks, let's implement a tiny RNN by hand to understand exactly what happens inside. This uses only NumPy (Python's numerical computing library) — no deep learning framework needed.

Python — Vanilla RNN from scratch (NumPy) 
import numpy as np

# ─────────────────────────────────────────────────────────────────
# VANILLA RNN — MANUAL IMPLEMENTATION
# Understanding what happens inside before using Keras/PyTorch
# ─────────────────────────────────────────────────────────────────

class VanillaRNN:
    """
    A simple Recurrent Neural Network built from scratch.
    
    Architecture:
      - Input size:   input_size  (number of features per time step)
      - Hidden size:  hidden_size (size of the memory/hidden state)
      - Output size:  output_size (number of values to predict)
    """
    
    def __init__(self, input_size, hidden_size, output_size):
        self.hidden_size = hidden_size
        
        # Initialize weights with small random values (Xavier initialization)
        scale = np.sqrt(2.0 / (input_size + hidden_size))
        
        # Weight matrix for input → hidden (shape: hidden_size × input_size)
        self.Wx = np.random.randn(hidden_size, input_size) * scale
        
        # Weight matrix for hidden → hidden (shape: hidden_size × hidden_size)
        self.Wh = np.random.randn(hidden_size, hidden_size) * scale
        
        # Bias for hidden layer (shape: hidden_size × 1)
        self.bh = np.zeros((hidden_size, 1))
        
        # Weight matrix for hidden → output (shape: output_size × hidden_size)
        self.Wy = np.random.randn(output_size, hidden_size) * scale
        
        # Bias for output layer (shape: output_size × 1)
        self.by = np.zeros((output_size, 1))
    
    def tanh(self, x):
        """Hyperbolic tangent — squishes values to range [-1, 1]"""
        return np.tanh(x)
    
    def forward(self, inputs):
        """
        Forward pass through the sequence.
        
        inputs: list of column vectors, each shape (input_size, 1)
        Returns: list of outputs, final hidden state
        """
        # Start with zero hidden state (no memory at the beginning)
        h_prev = np.zeros((self.hidden_size, 1))
        
        outputs = []
        self.hidden_states = [h_prev]  # Store for backprop
        self.inputs_cache = inputs      # Store for backprop
        
        for x_t in inputs:
            # ── CORE RNN EQUATION ──────────────────────────────────────
            # New hidden state = tanh(Wx * current_input + Wh * prev_hidden + bias)
            # This is where the "recurrence" happens!
            h_t = self.tanh(
                self.Wx @ x_t +    # Current input contribution
                self.Wh @ h_prev + # Previous memory contribution
                self.bh            # Bias
            )
            
            # Compute output from current hidden state
            y_t = self.Wy @ h_t + self.by
            
            outputs.append(y_t)
            self.hidden_states.append(h_t)
            h_prev = h_t  # Pass memory to next step
        
        return outputs, h_prev
    
    def predict_next(self, sequence):
        """Given a sequence, predict the next value."""
        outputs, final_hidden = self.forward(sequence)
        return outputs[-1]  # Return the last output as prediction


# ─────────────────────────────────────────────────────────────────
# DEMO: Predict next value in a sine wave sequence
# ─────────────────────────────────────────────────────────────────

# Generate a simple sequence: [0, 0.5, 0.87, 1.0, 0.87, 0.5, ...]
t = np.linspace(0, 2 * np.pi, 100)
sine_wave = np.sin(t)

# Create input sequences of length 5 and corresponding next values
seq_length = 5
sequences = []
targets = []

for i in range(len(sine_wave) - seq_length):
    seq = [np.array([[sine_wave[i + j]]]) for j in range(seq_length)]
    sequences.append(seq)
    targets.append(sine_wave[i + seq_length])

# Create the RNN
rnn = VanillaRNN(input_size=1, hidden_size=10, output_size=1)

# Test with the first sequence (untrained — random prediction)
test_seq = sequences[0]
prediction = rnn.predict_next(test_seq)

print(f"Sequence: {[round(s[0][0], 3) for s in test_seq]}")
print(f"True next value:     {round(targets[0], 4)}")

print(f"(Untrained — after training with backprop, it would learn the pattern)")
🟢 Building an LSTM with Keras (TensorFlow)

Keras (part of TensorFlow) provides a high-level API that lets us build LSTM networks with just a few lines of code. Each LSTM() layer handles all the gate computations automatically.

Python — LSTM for Sequence Prediction (Keras) 
import numpy as np
import tensorflow as tf
from tensorflow import keras
from tensorflow.keras import layers
from sklearn.preprocessing import MinMaxScaler
import matplotlib.pyplot as plt

# ─────────────────────────────────────────────────────────────────
# STEP 1: GENERATE SYNTHETIC SEQUENCE DATA
# (In real use, you would load actual data here)
# ─────────────────────────────────────────────────────────────────

np.random.seed(42)

# Simulate a noisy sine wave (like a trending financial series)
t = np.linspace(0, 50, 500)
data = np.sin(0.5 * t) + 0.1 * np.random.randn(500)

# ─────────────────────────────────────────────────────────────────
# STEP 2: NORMALIZE DATA
# MinMaxScaler scales values to range [0, 1]
# IMPORTANT: Fit ONLY on training data to prevent data leakage
# ─────────────────────────────────────────────────────────────────

scaler = MinMaxScaler(feature_range=(0, 1))
data_scaled = scaler.fit_transform(data.reshape(-1, 1))

# ─────────────────────────────────────────────────────────────────
# STEP 3: CREATE SLIDING WINDOW SEQUENCES
# window_size = how many past steps the LSTM sees to make a prediction
# ─────────────────────────────────────────────────────────────────

def create_sequences(data, window_size):
    """
    Convert a 1D time series into (input_sequence, target) pairs.
    
    Example with window_size=3, data=[1,2,3,4,5]:
      → ([1,2,3], 4), ([2,3,4], 5)
    """
    X, y = [], []
    for i in range(len(data) - window_size):
        X.append(data[i : i + window_size])
        y.append(data[i + window_size])
    return np.array(X), np.array(y)

WINDOW_SIZE = 30  # Use 30 past steps to predict next step
X, y = create_sequences(data_scaled, WINDOW_SIZE)

# ─────────────────────────────────────────────────────────────────
# STEP 4: TRAIN/TEST SPLIT — MUST BE CHRONOLOGICAL
# 80% for training, 20% for testing
# ─────────────────────────────────────────────────────────────────

split_idx = int(len(X) * 0.8)
X_train, X_test = X[:split_idx], X[split_idx:]
y_train, y_test = y[:split_idx], y[split_idx:]

# X shape: (samples, window_size, 1) — RNN needs 3D input
# The 3 dimensions are: (batch_size, timesteps, features)
X_train = X_train.reshape((X_train.shape[0], WINDOW_SIZE, 1))
X_test  = X_test.reshape((X_test.shape[0],  WINDOW_SIZE, 1))

print(f"Training samples:  {X_train.shape[0]}")
print(f"Test samples:      {X_test.shape[0]}")
print(f"Input shape:       {X_train.shape}  (samples, timesteps, features)")

# ─────────────────────────────────────────────────────────────────
# STEP 5: BUILD THE LSTM MODEL
# Architecture: 2 LSTM layers + Dense output layer
# ─────────────────────────────────────────────────────────────────

model = keras.Sequential([
    # LAYER 1: First LSTM layer
    # units=64 means 64 LSTM cells (each has all 4 gates)
    # return_sequences=True means output at every timestep (needed for stacking)
    layers.LSTM(
        units=64,
        return_sequences=True,
        input_shape=(WINDOW_SIZE, 1)  # (timesteps, features)
    ),
    
    # Dropout: randomly set 20% of neurons to 0 during training
    # This prevents overfitting (memorizing training data)
    layers.Dropout(0.2),
    
    # LAYER 2: Second LSTM layer
    # return_sequences=False means only output at the LAST timestep
    layers.LSTM(
        units=32,
        return_sequences=False
    ),
    
    layers.Dropout(0.2),
    
    # OUTPUT LAYER: Dense layer with 1 neuron = predict single value
    layers.Dense(units=1)
])

model.summary()  # Print architecture overview

# ─────────────────────────────────────────────────────────────────
# STEP 6: COMPILE AND TRAIN
# Adam optimizer: an adaptive learning rate optimizer
# MSE (Mean Squared Error): loss function for regression
# ─────────────────────────────────────────────────────────────────

model.compile(
    optimizer=keras.optimizers.Adam(learning_rate=0.001),
    loss='mean_squared_error'
)

# EarlyStopping: stop training if validation loss stops improving
# This prevents wasting time and overfitting
early_stop = keras.callbacks.EarlyStopping(
    monitor='val_loss',
    patience=10,    # Stop after 10 epochs of no improvement
    restore_best_weights=True
)

history = model.fit(
    X_train, y_train,
    epochs=100,
    batch_size=32,
    validation_split=0.1,
    callbacks=[early_stop],
    verbose=1
)

# ─────────────────────────────────────────────────────────────────
# STEP 7: EVALUATE AND PREDICT
# ─────────────────────────────────────────────────────────────────

predictions_scaled = model.predict(X_test)

# Inverse transform: convert scaled values back to original scale
predictions = scaler.inverse_transform(predictions_scaled)
y_test_actual = scaler.inverse_transform(y_test.reshape(-1, 1))

def engineer_features(df):

    # High value = strong directional day, Low = indecision (Doji)

    # Upper shadow: Distance from body top to High
    # Long upper shadow = selling pressure at high prices
    features['upper_shadow'] = df['High'] - df[['Open', 'Close']].max(axis=1)
    
    # Lower shadow: Distance from Low to body bottom
    # Long lower shadow = buying support at low prices (bullish)
    features['lower_shadow'] = df[['Open', 'Close']].min(axis=1) - df['Low']

    features['return'] = df['Close'].pct_change()
    
    # ── Trend features ──────────────────────────────────────────
    
    # Simple Moving Average ratio: Is price above its 20-day average?
    # Price / SMA20 > 1 means uptrend, < 1 means downtrend
    features['sma20_ratio'] = df['Close'] / df['Close'].rolling(20).mean()
    
    # Exponential Moving Average ratio (faster-responding than SMA)

    high_low = df['High'] - df['Low']

    # Rolling standard deviation of returns (20-day volatility)

    # Volume relative to its 20-day average
    # > 1 = higher than usual volume (significant move confirmation)
    features['volume_ratio'] = df['Volume'] / df['Volume'].rolling(20).mean()
    
    # Close price (normalized separately)
    features['close'] = df['Close']
    
    features = features.dropna()
    return features

🏗️ Complete Reusable Pipeline Class

This wraps everything into a clean, reusable Python class that you can use as a starting point for any time series project.

Python — Reusable RNN Time Series Pipeline 
import numpy as np
import tensorflow as tf
from tensorflow.keras import layers, models, callbacks
from sklearn.preprocessing import MinMaxScaler
from sklearn.metrics import mean_squared_error, mean_absolute_error

class TimeSeriesRNN:
    """
    A complete, reusable pipeline for time series forecasting.
    
    Supports: 'lstm', 'gru', or 'rnn' as model_type.
    
    Usage:
        pipeline = TimeSeriesRNN(model_type='lstm', window=30)
        pipeline.fit(time_series_array)
        predictions = pipeline.predict(n_steps=10)
    """
    
    def __init__(self, model_type='lstm', window=30, 
                 units=[64, 32], dropout=0.2):
        self.model_type = model_type
        self.window = window
        self.units = units
        self.dropout = dropout
        self.scaler = MinMaxScaler(feature_range=(0, 1))
        self.model = None
        self.is_fitted = False
    
    def _build_model(self, n_features=1):
        """Build the recurrent model based on chosen type."""
        
        # Select the recurrent layer class based on model_type
        layer_map = {
            'lstm': layers.LSTM,
            'gru':  layers.GRU,
            'rnn':  layers.SimpleRNN
        }
        RecurrentLayer = layer_map[self.model_type]
        
        model_layers = [
            # Input shape: (timesteps, features)
            RecurrentLayer(
                self.units[0],
                return_sequences=len(self.units) > 1,
                input_shape=(self.window, n_features)
            ),
            layers.Dropout(self.dropout)
        ]
        
        # Stack additional recurrent layers if provided
        for i, unit_count in enumerate(self.units[1:], 1):
            is_last = (i == len(self.units) - 1)
            model_layers.append(
                RecurrentLayer(unit_count, return_sequences=not is_last)
            )
            model_layers.append(layers.Dropout(self.dropout))
        
        model_layers.append(layers.Dense(1))
        
        self.model = models.Sequential(model_layers)
        self.model.compile(optimizer='adam', loss='mse')
        return self.model
    
    def fit(self, data, epochs=50, batch_size=32, verbose=1):
        """Prepare data and train the model."""
        data = np.array(data).reshape(-1, 1)
        
        # Scale data to [0, 1]
        data_scaled = self.scaler.fit_transform(data)
        
        # Create sliding window sequences
        X, y = [], []
        for i in range(len(data_scaled) - self.window):
            X.append(data_scaled[i:i + self.window])
            y.append(data_scaled[i + self.window])
        
        X = np.array(X)  # Shape: (n_samples, window, 1)
        y = np.array(y)
        
        # Build and train
        self._build_model(n_features=1)
        
        early_stop = callbacks.EarlyStopping(
            patience=10, restore_best_weights=True, monitor='val_loss'
        )
        
        self.model.fit(
            X, y, epochs=epochs, batch_size=batch_size,
            validation_split=0.1, callbacks=[early_stop], verbose=verbose
        )
        
        self.last_window = data_scaled[-self.window:]
        self.is_fitted = True
        print(f"✓ {self.model_type.upper()} model trained successfully!")
        return self
    
    def predict(self, n_steps=1):
        """
        Predict n_steps future values using the trained model.
        Uses rolling (autoregressive) prediction for multi-step.
        """
        if not self.is_fitted:
            raise ValueError("Call .fit() first!")
        
        current_window = self.last_window.copy()
        predictions_scaled = []
        
        for _ in range(n_steps):
            # Reshape for model: (1 sample, window_size, 1 feature)
            x_input = current_window.reshape(1, self.window, 1)
            pred = self.model.predict(x_input, verbose=0)[0][0]
            predictions_scaled.append(pred)
            
            # Slide window: drop oldest, add new prediction
            current_window = np.append(current_window[1:], [[pred]], axis=0)
        
        # Inverse transform: convert back to original scale
        predictions = self.scaler.inverse_transform(
            np.array(predictions_scaled).reshape(-1, 1)
        )
        return predictions.flatten()

📦 The Three TensorFlow / Keras Built-in Recurrent Layers
LayerFull ImportWhat It DoesWhen to Use
tf.keras.layers.SimpleRNN layers.SimpleRNN(units) Vanilla RNN — just the basic hₜ = tanh(Wx·xₜ + Wh·hₜ₋₁) equation, no gates Learning / baseline. Rarely used in production due to vanishing gradient.
tf.keras.layers.LSTM layers.LSTM(units) Full LSTM with forget, input, candidate, and output gates. Maintains both hidden state and cell state. Default choice for most sequence tasks. Excellent for long sequences.
tf.keras.layers.GRU layers.GRU(units) GRU with reset and update gates. Single hidden state. Fewer parameters than LSTM. When training speed matters or dataset is smaller. Often matches LSTM performance.
tf.keras.layers.RNN layers.RNN(cell) Base RNN class that wraps any custom cell object. You define the cell logic yourself. When you need a custom recurrent cell with non-standard update logic.
Python — Using tf.keras.layers.SimpleRNN, LSTM, and GRU 
import numpy as np
import tensorflow as tf
from tensorflow import keras
from tensorflow.keras import layers

# ─────────────────────────────────────────────────────────────────
# BUILT-IN KERAS RECURRENT LAYERS
#
# All three layers share the same API:
#   layers.SimpleRNN(units, ...)
#   layers.LSTM(units, ...)
#   layers.GRU(units, ...)
#
# Key shared parameters:
#   units            → number of recurrent cells (size of hidden state)
#   return_sequences → True: output at every timestep
#                      False: output only at the LAST timestep (default)
#   return_state     → also return the final hidden (and cell) state
#   dropout          → fraction of inputs to drop during training
#   recurrent_dropout→ fraction of recurrent connections to drop
#   go_backwards     → if True, process the sequence in reverse
# ─────────────────────────────────────────────────────────────────

# Dummy input: batch of 32 sequences, each 20 timesteps, 5 features
BATCH  = 32
STEPS  = 20   # timesteps (sequence length)
FEATS  = 5    # number of input features per timestep
dummy_input = np.random.randn(BATCH, STEPS, FEATS).astype("float32")


# ══════════════════════════════════════════════════════════════
# 1. tf.keras.layers.SimpleRNN
#    The plain vanilla RNN — useful to understand the baseline.
#    No gates, susceptible to vanishing gradient on long sequences.
# ══════════════════════════════════════════════════════════════

simple_rnn = layers.SimpleRNN(
    units=64,               # 64 hidden units
    activation="tanh",       # tanh is the default activation
    return_sequences=False,  # only return output at final timestep
    return_state=False,      # don't return hidden state separately
    dropout=0.0,             # no input dropout (set to e.g. 0.2 to regularize)
    recurrent_dropout=0.0   # no recurrent dropout
)

output_rnn = simple_rnn(dummy_input)
# output shape: (32, 64)  ← (batch_size, units)
print(f"SimpleRNN output shape: {output_rnn.shape}")


# ══════════════════════════════════════════════════════════════
# 2. tf.keras.layers.LSTM
#    Long Short-Term Memory — the workhorse of sequence modeling.
#    Has TWO states: hidden state h and cell state C.
#    return_state=True returns: output, final_h, final_C
# ══════════════════════════════════════════════════════════════

lstm_layer = layers.LSTM(
    units=64,
    return_sequences=False,  # set True if stacking another RNN on top
    return_state=True,       # return (output, hidden_state, cell_state)
    dropout=0.2,
    recurrent_dropout=0.1
)

# When return_state=True, LSTM returns THREE tensors:
lstm_out, state_h, state_c = lstm_layer(dummy_input)

# lstm_out  → output (same as state_h when return_sequences=False)
# state_h   → final hidden state  hₜ  (short-term memory)
# state_c   → final cell state    Cₜ  (long-term memory)
print(f"LSTM output shape:      {lstm_out.shape}")   # (32, 64)
print(f"LSTM hidden state (h):  {state_h.shape}")    # (32, 64)
print(f"LSTM cell state (C):    {state_c.shape}")    # (32, 64)


# ══════════════════════════════════════════════════════════════
# 3. tf.keras.layers.GRU
#    Gated Recurrent Unit — lighter than LSTM, often equally good.
#    Has only ONE state: hidden state h.
#    return_state=True returns: output, final_h
# ══════════════════════════════════════════════════════════════

gru_layer = layers.GRU(
    units=64,
    return_sequences=False,
    return_state=True,       # GRU returns (output, hidden_state) — no cell state
    dropout=0.2
)

gru_out, gru_state_h = gru_layer(dummy_input)

print(f"GRU output shape:       {gru_out.shape}")      # (32, 64)
print(f"GRU hidden state (h):   {gru_state_h.shape}")  # (32, 64)


# ══════════════════════════════════════════════════════════════
# 4. tf.keras.layers.RNN (base class with a custom cell)
#    Use this when you want to define your OWN recurrent logic
#    but still benefit from Keras's training infrastructure.
# ══════════════════════════════════════════════════════════════

class MyCustomCell(keras.layers.AbstractRNNCell):
    """
    A custom RNN cell that you define yourself.
    Keras wraps this with layers.RNN() to handle the sequence loop.

    Two methods you MUST implement:
      state_size  → size of the hidden state vector
      call(input, states) → one step forward pass
    """

    def __init__(self, units, **kwargs):
        super().__init__(**kwargs)
        self.units = units

    @property
    def state_size(self):
        return self.units  # one hidden state vector of size `units`

    def build(self, input_shape):
        # Create weight matrices (called once on first input)
        self.Wx = self.add_weight(
            shape=(input_shape[-1], self.units), initializer="glorot_uniform"
        )
        self.Wh = self.add_weight(
            shape=(self.units, self.units), initializer="orthogonal"
        )
        self.b  = self.add_weight(shape=(self.units,), initializer="zeros")
        self.built = True

    def call(self, inputs, states):
        # states is a list; states[0] is the previous hidden state hₜ₋₁
        h_prev = states[0]

        # Standard RNN equation: hₜ = tanh(Wx·xₜ + Wh·hₜ₋₁ + b)
        h_new = tf.nn.tanh(inputs @ self.Wx + h_prev @ self.Wh + self.b)

        # Return (output, [new_states]) — output is usually the new state
        return h_new, [h_new]


# Wrap the custom cell with the base RNN layer
custom_rnn = layers.RNN(
    MyCustomCell(units=32),
    return_sequences=False
)

custom_out = custom_rnn(dummy_input)
print(f"Custom RNN cell output: {custom_out.shape}")  # (32, 32)
🔍 The Most Confusing Parameter: return_sequences — Explained

This single parameter trips up nearly every beginner. Here is exactly what it does:

return_sequences = False (default)

Only the last timestep's output is returned. Use this for the final RNN layer when you want a single prediction (classification, regression).

return_sequences = True

Output at every timestep is returned. Use this when stacking multiple RNN layers — each layer needs the full sequence to pass to the next.

Golden Rule: All RNN layers except the last one must have return_sequences=True. The last RNN layer uses return_sequences=False (unless you need sequence-to-sequence output, like translation).
Python — Correctly Stacking Keras RNN Layers (Common Patterns) 
from tensorflow.keras import layers, models

# ──────────────────────────────────────────────────────────────
# PATTERN 1: Many-to-One (most common for classification/regression)
# Input:  sequence of shape (batch, timesteps, features)
# Output: single prediction per sequence
# ──────────────────────────────────────────────────────────────

many_to_one = models.Sequential([
    # Layer 1: LSTM — must use return_sequences=True to feed next LSTM
    layers.LSTM(128, return_sequences=True, input_shape=(60, 5)),
    layers.Dropout(0.2),

    # Layer 2: LSTM — last recurrent layer, return_sequences=False
    layers.LSTM(64, return_sequences=False),
    layers.Dropout(0.2),

    # Dense output — single value regression (e.g., next day's price)
    layers.Dense(1)
], name="many_to_one")

many_to_one.summary()


# ──────────────────────────────────────────────────────────────
# PATTERN 2: Many-to-Many (sequence-to-sequence, same length)
# Input:  sequence of T timesteps
# Output: sequence of T predictions (one per timestep)
# Use case: part-of-speech tagging, anomaly detection at each step
# ──────────────────────────────────────────────────────────────

many_to_many = models.Sequential([
    # ALL layers use return_sequences=True because we want output at every step
    layers.LSTM(64, return_sequences=True, input_shape=(60, 5)),
    layers.GRU(32,  return_sequences=True),

    # TimeDistributed applies a Dense layer to every timestep independently
    layers.TimeDistributed(layers.Dense(1))
    # Output shape: (batch, 60, 1) — one prediction per timestep
], name="many_to_many")


# ──────────────────────────────────────────────────────────────
# PATTERN 3: Bidirectional LSTM
# Reads the sequence both FORWARD and BACKWARD, then concatenates.
# Useful when you have access to the full sequence at once
# (not suitable for real-time/streaming predictions).
# ──────────────────────────────────────────────────────────────

bidirectional = models.Sequential([
    # layers.Bidirectional wraps any RNN layer
    # It runs two copies: one forward (left→right), one backward (right→left)
    # Outputs are concatenated: if LSTM has 64 units, output is 128
    layers.Bidirectional(
        layers.LSTM(64, return_sequences=True),
        input_shape=(60, 5)
    ),
    layers.Bidirectional(
        layers.LSTM(32, return_sequences=False)
    ),
    layers.Dense(1)
], name="bidirectional")


# ──────────────────────────────────────────────────────────────
# PATTERN 4: Functional API (for complex architectures)
# The Sequential API (above) only allows a linear stack of layers.
# For architectures with branches, skip connections, or multiple
# inputs/outputs, use the Functional API instead.
# ──────────────────────────────────────────────────────────────

inputs = keras.Input(shape=(60, 5), name="sequence_input")

# Two parallel paths — one LSTM, one GRU — then merge
x_lstm = layers.LSTM(64, return_sequences=False, name="lstm_path")(inputs)
x_gru  = layers.GRU(64,  return_sequences=False, name="gru_path")(inputs)

# Concatenate both paths
merged = layers.concatenate([x_lstm, x_gru], name="merge")

# Shared dense layers after the merge
x = layers.Dense(32, activation="relu", name="dense1")(merged)
x = layers.Dropout(0.2)(x)
output = layers.Dense(1, name="output")(x)

# Build the model from inputs to outputs
parallel_model = keras.Model(inputs=inputs, outputs=output, name="lstm_gru_parallel")
parallel_model.summary()

# Compile — same as always
parallel_model.compile(optimizer="adam", loss="mse")


# ──────────────────────────────────────────────────────────────
# PATTERN 5: Stateful RNN
# By default, Keras resets the hidden state at the start of each batch.
# With stateful=True, the state is carried over between batches.
# Use this when your sequence is too long to fit in one batch
# (e.g., a continuous stream of sensor data or audio).
# ──────────────────────────────────────────────────────────────

# For stateful RNNs, batch_size must be fixed and specified upfront
FIXED_BATCH = 1
stateful_model = models.Sequential([
    layers.LSTM(
        units=64,
        stateful=True,                              # carry state between batches
        batch_input_shape=(FIXED_BATCH, 50, 1),  # batch_size MUST be fixed
        return_sequences=False
    ),
    layers.Dense(1)
])

# You must manually reset state between independent sequences
# stateful_model.reset_states()  ← call this between unrelated sequences

print("\n✓ All patterns built successfully!")
print("\nSummary of when to use each:")
print("  Many-to-One:    Stock prediction, sentiment analysis")
print("  Many-to-Many:   Anomaly detection, seq-to-seq tagging")
print("  Bidirectional:  Text classification (full text available)")
print("  Functional API: Complex multi-input/output architectures")
print("  Stateful:       Streaming data, very long continuous sequences")
🔗 When and Why to Use return_state=True

Most of the time you don't need return_state=True. You need it in advanced architectures where you want to manually pass a state from one model into another. The most famous example is the Encoder-Decoder architecture used in machine translation.

Encoder-Decoder pattern: An encoder RNN reads the input sequence and produces a final state. That state is then used as the initial state of a decoder RNN that generates the output sequence. Without return_state=True, you cannot extract that final state to pass it across.
Python — return_state in an Encoder-Decoder Architecture 
import tensorflow as tf
from tensorflow import keras
from tensorflow.keras import layers
import numpy as np

# ─────────────────────────────────────────────────────────────────
# ENCODER-DECODER with return_state
#
# Task example: translate a sequence into another sequence
#   Encoder reads input  → produces a "context" (the final state)
#   Decoder starts from that context → generates the output
# ─────────────────────────────────────────────────────────────────

LATENT_DIM  = 64   # size of the shared hidden state
ENC_STEPS   = 20   # encoder input length
DEC_STEPS   = 15   # decoder output length
FEATS       = 8

# ── ENCODER ─────────────────────────────────────────────────────

encoder_inputs = keras.Input(shape=(ENC_STEPS, FEATS), name="encoder_input")

# return_state=True → returns (output, final_h, final_c)
# We only care about the states here, not the per-step output
encoder_lstm = layers.LSTM(LATENT_DIM, return_state=True, name="encoder_lstm")
_, enc_h, enc_c = encoder_lstm(encoder_inputs)
# enc_h → final hidden state  (encodes meaning of the full input)
# enc_c → final cell state    (long-term memory from the input)

encoder_states = [enc_h, enc_c]   # package both states together


# ── DECODER ─────────────────────────────────────────────────────

decoder_inputs = keras.Input(shape=(DEC_STEPS, FEATS), name="decoder_input")

decoder_lstm = layers.LSTM(
    LATENT_DIM,
    return_sequences=True,  # we need output at every decoder step
    return_state=True,
    name="decoder_lstm"
)

# Pass encoder_states as the INITIAL STATE of the decoder
# This is how the decoder "knows" what the encoder read
decoder_outputs, _, _ = decoder_lstm(
    decoder_inputs,
    initial_state=encoder_states   # ← the key line!
)

# Dense layer applied to every decoder timestep
decoder_dense = layers.TimeDistributed(
    layers.Dense(1), name="output_projection"
)
final_output = decoder_dense(decoder_outputs)

# Build full encoder-decoder model
seq2seq = keras.Model(
    inputs=[encoder_inputs, decoder_inputs],
    outputs=final_output,
    name="encoder_decoder"
)

seq2seq.compile(optimizer="adam", loss="mse")
seq2seq.summary()

# Quick test with dummy data
enc_data = np.random.randn(16, ENC_STEPS, FEATS)
dec_data = np.random.randn(16, DEC_STEPS, FEATS)
targets  = np.random.randn(16, DEC_STEPS, 1)

output = seq2seq([enc_data, dec_data])
print(f"\nEncoder input shape:  {enc_data.shape}")
print(f"Decoder input shape:  {dec_data.shape}")
print(f"Final output shape:   {output.shape}")   # (16, 15, 1)
print("\n✓ Encoder-Decoder built! The encoder's final state")
print("  was passed as the decoder's initial state via return_state.")

🚨 Common Mistakes and How to Avoid Them
MistakeWhy It's WrongFix
Random train/test split on time series Future data leaks into training — the model "cheats" Always split chronologically: train on early data, test on later data
Fitting the scaler on all data The scaler "sees" future values during normalization Fit scaler on training set only; transform test set with same scaler
Forgetting to reshape for RNN input Keras LSTM requires 3D input: (samples, timesteps, features) Use X.reshape(X.shape[0], window, n_features)
Using return_sequences=False between LSTM layers The second LSTM layer receives only one vector, not a sequence Set return_sequences=True for all but the last RNN layer
Not inverse transforming predictions Results are in [0,1] scale, meaningless to end users Always call scaler.inverse_transform() on predictions

Test Your Understanding

🧠 In an LSTM network, what is the purpose of the Cell State (Cₜ)?
✓ Correct! The Cell State is the "long-term memory highway" of the LSTM. It runs through the network with only minor linear interactions (forget + add), allowing gradients and information to flow over many time steps. This is why LSTM solves the vanishing gradient problem that plagued vanilla RNNs.
🧠 When creating sequences for a time series model, why must we split the data chronologically (not randomly)?
✓ Exactly! This is called "data leakage." If we split randomly, the training set might include a window ending on day 200 with a target from day 201 — but the test set might include a sequence starting on day 190. The model would have already "seen" the future during training. Chronological splitting ensures the model is always predicting truly unseen future data.
🧠 A GRU uses only 2 gates while LSTM uses 4. What is the main practical advantage of GRU?
✓ Right! GRU's main advantage is efficiency. With ~25% fewer parameters than an equivalent LSTM, it trains faster and generalizes better on smaller datasets. In practice, GRU often performs similarly to LSTM on medium-length sequences. The choice between them is mostly a hyperparameter tuning decision — try both and pick the one that performs better on your validation set.
